# cppcheck-suppressions.txt
#
# Authoritative registry of every cppcheck suppression in the linkdb
# CI pipeline.  Spec-0.27.5 §3.4 (suppression maintenance rules)
# requires every suppression to have a one-line "Reason:" comment
# explaining why the cppcheck warning is wrong or unactionable.
#
# Implementation note: the suppressions are NOT loaded from this file
# by scripts/ci/run-cppcheck.sh -- they are passed as --suppress= CLI
# flags directly because cppcheck 2.13's --suppressions-list parser
# rejects mixed-comment files with "Failed to add suppression. No id."
# even when valid entries are present.  This file is kept as the
# human-facing registry; run-cppcheck.sh is the executable copy.
# Whenever you add a suppression, update both files in lockstep.
#
# Spec:    pgrac/specs/spec-0.27.5-static-analysis.md
# Design:  pgrac/docs/ci-static-analysis.md
# Baseline: pgrac/docs/ci-static-analysis-baseline-stage0.27.md
#
# Format: <id>[:<file>[:<line>]]   id is a cppcheck rule name.

# === Global PG framework noise (apply to all files) ===
# Reason: cppcheck flags every PG SRF function as "never used"
# because it cannot resolve the PG_FUNCTION_INFO_V1 macro registration.
unusedFunction
# Reason: cppcheck cannot resolve PG's hand-rolled include layout
# reliably with -I; missingInclude noise is not actionable.
missingInclude
missingIncludeSystem
# Reason: future inline /* cppcheck-suppress xxx */ removals should
# not fire unmatched-suppression warnings during the transition.
unmatchedSuppression

# === PG-upstream headers reachable via -I src/include ===
# Reason: cppcheck transitively follows -I src/include and warns on
# PG-upstream headers; spec-0.27.5 §1.2 limits scope to cluster code.
# PG header s_lock.h emits #error for unsupported-platform builds;
# cppcheck cannot evaluate the platform conditionals correctly.
preprocessorErrorDirective:src/include/storage/s_lock.h
# Reason: PG-upstream defensive null check; pg_list.h is not pgrac code.
nullPointerRedundantCheck:src/include/nodes/pg_list.h
# Reason: PG-upstream defensive null check; bufpage.h is not pgrac code.
nullPointerRedundantCheck:src/include/storage/bufpage.h
# Reason: PG-upstream xlogreader.h reachable via cluster_smgr.c -> smgr.h
# transitive include (added in stage 1.2); not pgrac code.
constParameterPointer:src/include/access/xlogreader.h
